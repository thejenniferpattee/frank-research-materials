# Security Policy

## Scope
This repository is research and prototype code and documentation. It is not a clinical tool and is not intended for production deployment.

## Supported Versions
Only the default branch is maintained. Older commits and forks may be unpatched.

## Reporting a Vulnerability
If you believe you have found a security vulnerability, please report it privately.

**Send a report to:** thejenniferpattee@gmail.com  
Include:
- A short description of the issue
- Affected files or endpoints
- Steps to reproduce
- Any proof of concept needed to validate the issue

### Please Do Not
- Open a public GitHub issue for active vulnerabilities  
- Include real participant data, personal health information, or private credentials  
- Run automated scans against domains or services you do not own or have permission to test

## What You Can Expect
- Acknowledgement within 3 business days  
- A best effort fix and public note once a patch is available, or a documented mitigation if a full fix is not feasible

## Disclosure
Coordinated disclosure is appreciated. If you need a public credit line, include the name or handle you want listed.
